# Running locally

## Main application (Vite)

From the **repository root**:

```bash
npm run dev
```

- **Bundler:** Vite (`vite.config.js`)  
- **Default URL:** `http://localhost:5173` — if the port is busy, Vite chooses the next free port; watch the terminal output.  
- **Hot reload:** Enabled for React components and styles.

### Production-like preview

```bash
npm run build
npm run preview
```

Serves the optimized build locally for smoke testing.

## Documentation site (Docusaurus)

From the **repository root**:

```bash
npm run docs:dev
```

Or:

```bash
cd website
npm run start
```

- **Default URL:** `http://localhost:3000`  
- **Source:** Markdown/MDX under `website/docs/`  

### Build static docs

```bash
npm run docs:build
# output: website/build/
npm run docs:serve   # optional local preview of the build
```

## Running backend + frontend together

Typical workflow:

1. Start your API at the URL in `VITE_BACKEND_URL` (example: `http://localhost:5050`).  
2. Ensure `.env` points to that URL.  
3. Start Firebase-dependent flows only after Firebase env vars are correct.  
4. Run `npm run dev` for the SPA.

## Troubleshooting

| Symptom | Things to check |
|--------|-------------------|
| Blank API responses / CORS errors | Backend must allow your Vite origin; configure CORS on the API. |
| Login works but no user profile | `/auth/me` or `/auth/token` failing — inspect Network tab and backend logs. |
| Stuck on “Loading…” | `UserContext` waiting on Firebase + backend; check `isLoading` paths in `ProtectedRoutes`. |
