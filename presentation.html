01 · Purpose

The problem we set out to solve.

Small nonprofits collect donations through Stripe, but the data lives in one place while the work of thanking donors lives in another. Acknowledging donors becomes slow, manual, and error-prone.

Donor data is scattered
Payments sit in Stripe with no clean, searchable view of who gave, how much, and when.
Thank-yous & receipts are manual
Letters and receipts are written by hand for every donor, which is tedious and easy to miss.
No visibility into impact
No easy way to see trends, top donors, or whether giving is growing week over week.
02 · Spec Overview

What the platform does.

🔐 Secure admin access

Authentication with sign-up, login, password reset, and role-based routes that keep the platform restricted to admins.

↻ Import from Stripe

One-click sync pulls donations & donor details from Stripe; refreshed daily so the dashboard is always current.

🧾 Auto-generated documents

Receipts and letter templates are generated automatically from each donation's data.

✉️ Send directly from the platform

Send personalized thank-yous & receipts to one donor, or to many at once with bulk send.

🔎 Search, filter & sort

Browse donations and donors with search, filters, and sorting by amount, date, and more.

📈 Visualizations & stats

Charts of donations over time, plus growth rate, new donors, and totals to measure impact.

02 · Spec Overview

Core goals vs. stretch goals.

Core goals

The must-haves we committed to
  • Admin-only platform with secure authentication
  • Auto-generate letter templates & receipts after a monetary donation
  • View donor data with search, filters & visualizations

Stretch goals

Reaches beyond the core scope
  • Manually enter non-monetary donations from vendors & partners (e.g. food) ✓ done
  • Send document emails automatically or directly through the platform ✓ done
  • Unified site for database and volunteer management future

Non-goal: the platform has no donor-facing view, so donors can't access it.

02 · Spec Overview

Edge cases we designed for.

Daily sync without duplicates

Re-importing from Stripe reconciles against existing records so the same donation never lands twice.

"Already sent?" flags

Each donation tracks whether a thank-you / receipt has gone out, so donors never get double-emailed.

Bulk send, partial failure

Sending to many donors at once reports exactly which succeeded and which need a retry.

Monetary vs. non-monetary

Food & in-kind gifts have no Stripe charge, so they're entered manually and handled separately from card donations.

Access control

Non-admins hitting a protected route are routed to a no-access page rather than seeing data.

04 · Challenges

What was hard, and how we solved it.

Messy Stripe data → clean records

What we did: normalize partial or missing donor fields (names, addresses) from Stripe into one consistent donor model the rest of the app can rely on.

Learning AWS deployment & CDK

What we did: got up to speed on AWS and defined our infrastructure as code with the AWS CDK, so the backend deploys the same way every time.

Charts that adapt to any date range

What we did: auto-bucket donations by day, week, or month based on the selected range, so the trend line stays readable from one week to a full year.

Deploying across three services

What we did: wired Firebase hosting, the AWS backend, and Resend email into one documented pipeline so deploys are repeatable.
